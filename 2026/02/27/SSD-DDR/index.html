SSD固态硬盘

发表于 2026-02-27 标签： # 行业体感

在车上，想要弄一个存储空间大一些的SSD固态硬盘来存行车记录仪和哨兵模式下产生的视频，这样记录的视频被覆盖重写的周期会长一些。在小红书上查了一下，点赞评论数较多的是买的SSD固态盘加硬盘盒，图方便省事可以直接上移动固态硬盘盒或者三星迷你U盘fit plus 512G，最好根据自己车的品牌型号提前在网上收集一下其他人遇到的坑。

作为普通消费者，根据自己的预算和需求主要关注三个点：SSD固态硬盘PCIe的版本、颗粒是TLC的还是QLC、是否原厂。

听网友说PCIe4.0遇到过发烫降速，SSD固态硬盘选了PCIe3.0，考虑环境温度、使用场景和安装场景的稳定性，选择了TLC颗粒，原厂JD官方旗舰店购买。

我自己最后是买了一个前身东芝存储器的铠侠RC20 1TB PCIe3.0 1GB缓存（想支持国货，没货了，就很离谱），固态硬盘盒是佳翼NVMe 583 车载专用款，然后是买了3M家的20x60的魔术贴来固定硬盘，总共花费1032.52元（有点小贵了较年前）。

SSD固态硬盘： 在电脑上格式化选exFAT，（这里有一个在win10上格式化分区的具体操作链接），网上说预留部分空间(OP)不分区，防止硬盘快写满时发生降速异常，没有必要去浪费存储空间，我们不是7*24小时高强度写入的服务器场景。可以在自己的车机系统上去试一下不分格外的空闲空间，实践出真知。插在车上时，再在车机上进行格式化一下。

硬盘盒： 问卖家要重写驱动的程序，关掉自动休眠（设置自定义休眠时间为0）

硬盘固定： 根据硬盘盒尺寸选购魔术贴来固定

先来说一下固态硬盘的价格，由于这波AI浪潮，导致存储行业个人消费电子存储产能受限拉高了存储设备的价格，至少是在原来的基础上翻倍不止。希望中国芯片原厂与制造商长江存储（YMTC）和长鑫存储（CXMT）能利用这次机会再进一步发展，将价格稳下来，成为全球行业有话语权的玩家。

  • 长江存储：国内唯一的3D NAND闪存原厂，自主研发Xtracjing架构，232层技术达到国际一线水平，2025年全球市占率约10%。（来自：Deepseek）
  • 长鑫存储：国内DRAM(内存)芯片龙头，已实现DDR4/DDR5量产，2025年全球市占率预计突破5%。（来自：Deepseek）

TLC颗粒和QLC颗粒的区别：

最主要的区别在每个单元存储的位数，一个是三位一个是四位。

  • TLC：Triple-Level Cell，三层单元
  • QLC：Quad-Level Cell，四层单元
对比维度 TLC QLC
每个单元存储位数 3位 4位
存储密度 较高 最高 (同样物理空间容量更大)
制造成本 较低 最低
理论擦写寿命 约 1000-3000 次 约 100-1500 次
读写速度 快 较快 (但大文件写入易掉速)
数据保持性 好 一般

PCIe版本间的区别：

最主要的区别在于传输速度，每一代是前一代的两倍。

对比维度 PCIe 3.0 PCIe 4.0 PCIe 5.0
通俗比喻 双向8车道高速 双向16车道高速 双向32车道高速
传输速率 8 GT/s 16 GT/s 32 GT/s
理论带宽 (x4) 约 4 GB/s 约 8 GB/s 约 16 GB/s
实际速度 (顺序读写) 约 3500 MB/s 约 7000 - 7500 MB/s 约 10000 - 14000+ MB/s
主流定位 性价比之选，满足日常 高性能游戏/办公主流 极致生产力，专业创作

SSD固态硬盘配缓存：

我买的1TB的硬盘配了1GB的缓存，可以方便在车机系统中视频快速写入和回放。

在SSD（固态硬盘）语境下，通常是指1GB的独立DRAM缓存。对于1TB容量的SSD来说，这个1:1000的配比是一个非常经典的方案。

它最核心的使命是：

  • 快速寻址：SSD读写数据时需要查一张“地址地图”（FTL映射表）。有缓存就可以把这整张地图放进去，主控芯片查路线的速度极快。
  • 避免卡顿：如果没独立缓存，地图就只能存在慢得多的闪存里。这会导致日常打开软件、小文件读写（即4K随机性能）变慢，SSD越用越卡的可能性也更高。

MBR 与 GPT 分区表：Windows 与 Linux 支持度对比

下表详细列出了 MBR（主引导记录）和 GPT（GUID 分区表）在 Windows 和 Linux 系统中的支持情况、限制及适用场景。

特性 MBR (主引导记录) GPT (GUID 分区表)
硬盘容量支持 最大 2TB。超过 2TB 的部分无法识别和使用。 无上限（理论最大 18EB）。完美支持 2TB 及以上的大容量硬盘。
分区数量 最多 4 个主分区，或 3 个主分区 + 1 个扩展分区（扩展分区内可划分多个逻辑驱动器）。 最多 128 个主分区（Windows 下）。Linux 下同样支持多达 128 个分区，没有扩展/逻辑分区的概念。
系统引导模式 传统 BIOS 模式（也称 Legacy 模式）。 现代 UEFI 模式（必需）。注：Linux 可通过特殊配置在 BIOS 模式下从 GPT 磁盘启动，但 Windows 不行。
数据安全与恢复 较差。分区表仅有一份，存储在磁盘头部，容易因病毒或写入错误导致损坏，且难以恢复。 优秀。分区表在主次两端有多份备份，具有 CRC32 校验和容错机制，损坏后可自动从备份恢复。
操作系统兼容性 极高。所有版本的 Windows（从 95 到 11）和 Linux 均可识别和使用 MBR 磁盘。 良好。Windows 要求：仅 Vista/Server 2008 及之后版本支持作为数据盘；仅 64位 Windows 且配备 UEFI 才支持从 GPT 启动。Linux 要求：现代发行版（内核 2.6.38 以上）完美支持，包括启动和数据盘。
引导加载器 Windows: bootmgr (位于活动分区) + boot.ini 或 BCD 存储。
Linux: GRUB Legacy 或 GRUB2 (安装于 MBR 和分区引导扇区)。		 Windows: bootmgfw.efi (位于 ESP - EFI 系统分区) + BCD 存储。
Linux: GRUB2 (针对 UEFI 编译，安装于 ESP)、systemd-boot 或其他 UEFI 引导器。
适用场景 • 老旧电脑（仅支持 BIOS 启动）。
• 小容量硬盘（≤ 2TB）。
• 需要安装 32位 Windows 或极老版本 Windows。
• 多系统兼容性要求极高的环境。		 • 现代电脑（2012 年后出厂，预装 Win8/10/11）。
• 大容量硬盘（> 2TB）。
• 需要 UEFI 安全启动（Secure Boot）功能。
• 希望获得更快的启动速度和更好的数据安全性。
• Linux 服务器或桌面（推荐首选）。

🔍 如何查看当前磁盘的分区表类型？

Windows 系统
  1. 右键点击“开始”按钮（或按 Win + X），选择“磁盘管理”。
  2. 在磁盘管理窗口底部，右键点击目标磁盘的标签（如“磁盘 0”），选择“属性”。
  3. 切换到“卷”选项卡，查看“磁盘分区形式”一栏，会显示是“主引导记录(MBR)”还是“GUID 分区表(GPT)”。

NTFS vs. exFAT 核心对比

特性维度 NTFS exFAT
最佳适用场景 Windows 系统盘、内部硬盘、游戏盘、企业服务器 U盘、SD卡、移动硬盘，需在 Windows 和 Mac 间传输文件时
兼容性 Windows完美支持；Mac默认只能读，无法写入；Linux部分发行版需安装驱动 跨平台王者，Windows/macOS/Linux 均可直接读写，无需额外设置
关键功能 功能全面：支持文件加密、权限设置、磁盘配额、日志功能（可恢复数据） 功能精简：为了提高闪存设备效率和寿命，舍弃了NTFS的高级功能
安全性 高。日志功能可避免因断电或意外拔出导致的数据损坏，支持文件加密（EFS） 低。缺乏日志功能，突然断电或直接拔出，数据损坏或丢失的风险更高
理论限制 文件大小：16EB；分区大小：256TB 文件大小：16EB；分区大小：128PB
